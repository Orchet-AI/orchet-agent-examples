# Orchet Weather Public Readonly Agent

Orchet specialist agent for Weather Public Readonly. Get current public weather for a city without user authentication.

Generated by [@orchet/agent-cli](https://github.com/Orchet-AI/orchet-agent-cli). Edit `lib/vendor-client.ts` to plug in real Weather Public Readonly API calls, then ship.

## Endpoints

| Endpoint | Surface |
|---|---|
| `GET /.well-known/agent.json` | Manifest the Orchet registry polls |
| `GET /openapi.json` | OpenAPI 3.1 spec for the tools |
| `GET /health` | Liveness probe |
| `POST /tools/<name>` | One per generated tool — see `app/tools/` |

## Local dev

```bash
cp .env.example .env.local          # set optional local overrides
npm install
npm test                            # typecheck + manifest + local E2E artifact validation
npm run validate-manifest
orchet-agent dev                    # serves on :3007
```

## Deploy

```bash
vercel link

vercel env add ORCHET_WEATHER_PUBLIC_READONLY_AGENT_BASE_URL production    # e.g., https://weather-public-readonly.orchet.ai
vercel deploy --prod
```



## Submit to the Orchet Store

Once `weather-public-readonly.orchet.ai` is live:

```bash
export ORCHET_DEVELOPER_TOKEN=orchet_dev_...
export ORCHET_CONTACT_EMAIL=you@example.com

npm run e2e:submit -- \
  --agent-base https://weather-public-readonly.orchet.ai
```

The script validates the deployed manifest, OpenAPI, and health endpoint,
submits the hosted SDK/API agent to Orchet, polls submission status, and prints:

```text
submission accepted: <submission_id>
state: pending
developer url: https://www.orchet.ai/developer/submissions/<submission_id>
admin review url: https://www.orchet.ai/admin/review-queue
```

To validate the deployed endpoints without submitting:

```bash
npm run e2e:submit -- \
  --agent-base https://weather-public-readonly.orchet.ai \
  --validate-only
```

The lower-level CLI command is still available when you need to submit manually:

```bash
ORCHET_DEVELOPER_TOKEN=... orchet-agent submit \
  --manifest-url https://weather-public-readonly.orchet.ai/.well-known/agent.json \
  --openapi-url https://weather-public-readonly.orchet.ai/openapi.json \
  --health-url https://weather-public-readonly.orchet.ai/health \
  --contact-email you@example.com
```

Signed bundles are optional advanced metadata for verified releases:

```bash
ORCHET_SIGNING_SECRET=... orchet-agent sign --bundle ./bundle.tgz --out .orchet/signature.json
ORCHET_DEVELOPER_TOKEN=... orchet-agent submit \
  --manifest-url https://weather-public-readonly.orchet.ai/.well-known/agent.json \
  --openapi-url https://weather-public-readonly.orchet.ai/openapi.json \
  --health-url https://weather-public-readonly.orchet.ai/health \
  --bundle ./bundle.tgz \
  --signature-file .orchet/signature.json \
  --contact-email you@example.com
```
